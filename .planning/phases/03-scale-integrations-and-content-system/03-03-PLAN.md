  <name>Task 1: Create city-priority matrix generation script</name>
  <files>scripts/city-priority-matrix.ts</files>
  <action>
    Create `scripts/city-priority-matrix.ts` that queries Sanity and generates a CSV:

    1. Query all cities from Sanity with GROQ:
       ```
       *[_type == "city"] {
         title, "slug": slug.current, tier, population, medianRent, vacancyRate,
         "province": province->title,
         "provinceSlug": province->slug.current,
         "country": province->country,
         "serviceCount": count(*[_type == "cityService" && city._ref == ^._id]),
         "categoryCount": count(*[_type == "propertyCategory" && city._ref == ^._id]),
         "listingCount": count(*[_type == "propertyListing" && city._ref == ^._id])
       } | order(tier asc, population desc)
       ```

    2. Calculate a priority score per city based on: tier (weight 40%), population (weight 30%), service coverage completeness (weight 30%)
       - Tier 1 = 40 points, Tier 2 = 20 points, Tier 3 = 10 points
       - Population normalized: (population / max_population) * 30
       - Service coverage: (serviceCount / 8) * 30 (8 = total services)

    3. Output CSV to `.planning/content/city-priority-matrix.csv` with columns:
       rank, city, province/state, country, tier, population, slug, provinceSlug, url, priorityScore, servicePages, categoryPages, listingPages, status

    4. Status values: "complete" (has all 8 services + 4 categories), "partial" (has some), "stub" (city exists, no service pages)

    5. Use @sanity/client directly (read-only, no write token needed)
    6. Run via: `npx tsx scripts/city-priority-matrix.ts`
    7. Import dotenv for project ID / dataset from env
  </action>
  <verify>npx tsc --noEmit --skipLibCheck scripts/city-priority-matrix.ts 2>&1 | head -10</verify>
  <done>Script compiles and when run generates the CSV matrix file</done>
</task>
