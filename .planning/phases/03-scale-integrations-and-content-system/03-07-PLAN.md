    # Content Publishing Workflow

    ## Prerequisites
    - Sanity Studio access (login at /studio)
    - Content brief or AI-generated draft ready
    - Local data researched (population, median rent, vacancy rate, neighbourhoods)

    ## Workflow A: Publishing a New City

    ### Step 1: Create the City Document
    1. In Sanity Studio, click "+ New Document" > "City"
    2. Fill ALL required fields (these are enforced by the schema):
       - Title: City name (e.g., "Mississauga")
       - Slug: Auto-generated from title (e.g., "mississauga")
       - Province: Select the parent province/state
       - Tier: Select 1 (priority), 2 (standard), or 3 (long-tail)
       - Population: Current population number
       - Median Rent: Monthly median rent in local currency
       - Vacancy Rate: Current vacancy rate as percentage
       - Neighbourhood Names: Minimum 3 neighbourhoods (REQUIRED -- prevents thin content)
       - Description: 2-3 sentences about the city's rental market
    3. SEO Section:
       - Meta Title: "[City] Property Management | MoveSmart Rentals" (50-60 chars)
       - Meta Description: "Professional property management in [City], [Province]. ..." (140-160 chars)
    4. Publishing Controls:
       - Check "Include in Sitemap"
       - Leave "No Index" unchecked
    5. Click "Publish"

    ### Step 2: Create CityService Documents (8 per city)
    For each of the 8 services (tenant-placement, tenant-screening, rent-guarantee, leasing-services, rental-preparation, rental-marketing, portal-technology, pricing):
    1. Click "+ New Document" > "City Service"
    2. Fill required fields:
       - City: Select the city created in Step 1
       - Service: Select the service
       - Province: Select the province
       - Hero Headline: Unique headline for this city+service (NOT just "[Service] in [City]")
       - Local Body: 200+ words of locally-specific content
         - MUST include local regulatory references
         - MUST mention specific neighbourhoods
         - MUST include local market data
       - Local Regulatory: Province/state-specific compliance info
       - FAQ: 3-5 locally relevant questions and answers
    3. SEO section with unique meta title and description
    4. Click "Publish"

    ### Step 3: Verify the Page is Live
    1. Wait 60 seconds (ISR revalidation via webhook)
    2. Navigate to /ca/{province}/{city}/ (or /us/{state}/{city}/)
    3. Verify: city hub page shows local data, services grid, property types
    4. Navigate to /ca/{province}/{city}/{service}/
    5. Verify: CityService page shows all content sections

    ## Workflow B: Publishing a Blog/Guide

    ### Step 1: Create Blog/Guide Document
    [Similar step-by-step for blog content...]

    ## Quality Checklist (Before Publishing)
    - [ ] All required fields are filled (Sanity will show validation errors if not)
    - [ ] Meta title is 50-60 characters
    - [ ] Meta description is 140-160 characters
    - [ ] Neighbourhood names have 3+ entries (anti-thin-content requirement)

    - [ ] FAQ has 3-5 questions with substantive answers
    - [ ] "Include in Sitemap" is checked

       ```markdown
       # Weak Page Refresh Workflow

       ## When to Refresh
       Run this workflow monthly after GSC data accumulates (minimum 28 days of data).

       ## Step 1: Identify Weak Pages
       In Google Search Console > Performance:
       1. Set date range to last 28 days
       2. Filter pages by your domain
       3. Sort by Impressions (descending)
       4. Identify pages with HIGH impressions but LOW CTR (<2%)
         - These pages are being shown in search but not clicked
       5. Also identify pages with declining average position (was <10, now >20)

       ## Step 2: Diagnose the Issue
       For each weak page, check:
       - Meta title: Is it compelling? Does it match search intent?
       - Meta description: Does it include a call to action?
       - Content depth: Does the page have 500+ words of unique content?
       - Local specificity: Does it mention specific neighbourhoods, data points?
       - Schema markup: Is FAQ schema present?
       - Internal links: Does the page link to and from related pages?

       ## Step 3: Refresh Priority Matrix
       | Metric | Action |
       |--------|--------|
       | High impressions, low CTR | Rewrite meta title and description |
       | Declining position | Add more content, improve local data |
       | No impressions | Check indexing, add internal links |
       | High bounce rate | Improve page content and CTA placement |

       ## Step 4: Update Content in Sanity
       1. Open the page's CMS document in Sanity Studio
       2. Update the identified weak areas
       3. Publish
       4. Wait for ISR revalidation (60 seconds)
       5. Verify changes on the live page

       ## Step 5: Track Results
       After refresh, wait 2-4 weeks and compare:
       - CTR change
       - Average position change
       - Impressions change
       Record in the content status tracker.

       ## Refresh Frequency
       - Tier-1 cities: Monthly review
       - Tier-2 cities: Quarterly review
       - Blog/guides: Quarterly review
       ```
