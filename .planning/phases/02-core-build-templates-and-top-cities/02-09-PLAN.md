    **sitemap({ id }):** Switch on id to return URLs:

    **'static' segment:**
    Return hardcoded URLs for all static pages:
    - / (homepage) -- priority 1.0, changeFrequency 'weekly'
    - /owners/ -- priority 0.9, 'weekly'
    - /tenants/ -- priority 0.9, 'weekly'
    - /services/ -- priority 0.8, 'weekly'
    - /locations/ -- priority 0.8, 'weekly'
    - /pricing/ -- priority 0.7, 'monthly'
    - /resources/ -- priority 0.7, 'weekly'
    - /franchising/ -- priority 0.6, 'monthly'
    - /about/ -- priority 0.5, 'monthly'
    - /contact/ -- priority 0.5, 'monthly'
    - /faq/ -- priority 0.6, 'weekly'
    All with lastModified: new Date()
    Use NEXT_PUBLIC_SITE_URL for base URL.

    **'ca-cities' segment:**
    Fetch provinces and cities from CMS:
    - Province pages: /ca/{province}/ -- priority 0.7
    - City hub pages: /ca/{province}/{city}/ -- priority 0.8 for Tier-1, 0.6 for others
    Query: `*[_type == "city"]{ slug, tier, "provinceSlug": province->slug.current }` with tags ['city']

    **'ca-services' segment (largest):**
    Fetch all CityService combinations:
    - CityService pages: /ca/{province}/{city}/{service}/ -- priority 0.9 for Tier-1 cities, 0.7 for others
    - Also include service hub pages: /services/{service}/ -- priority 0.8
    Query: use CITY_SERVICE_PARAMS_QUERY pattern + city tier. Tags: ['cityService', 'service']

    **'blog' segment:**
    Fetch all blog/guide documents:
    - /resources/{slug}/ -- priority 0.6, changeFrequency 'weekly'
    Query from blogGuide documents. Tags: ['blogGuide']

    **'listings' segment:**
    Fetch property categories and listings:
    - Property category: /ca/{province}/{city}/{propertyType}-for-rent/ -- priority 0.7
    - Property detail: /ca/{province}/{city}/rentals/{slug}/ -- priority 0.5
    Query from propertyCategory and propertyListing. Tags: ['propertyCategory', 'propertyListing']

    **'resources' segment:**
    Fetch comparisons and case studies:
    - /resources/{slug}/ -- priority 0.6
    Query from comparison and caseStudy. Tags: ['comparison', 'caseStudy']

    For ALL URLs:
    - Prepend siteUrl (from NEXT_PUBLIC_SITE_URL || 'https://movesmartrentals.com')
    - Ensure trailing slash on all URLs
    - Set lastModified to new Date() (will be more accurate when CMS _updatedAt is available)
    - Cast changeFrequency with `as const` for TypeScript
    - Return empty array [] for segments with no CMS data (not an error)
